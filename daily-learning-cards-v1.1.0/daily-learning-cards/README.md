# daily-learning-cards

> 一个自动化的"学习秘书"，帮你把零散对话变成可复习的知识卡片。

---

## 💡 总结

**这个 Skill 做了 4 件核心事：**

1. **记录** → 把我们的对话写成结构化记忆
2. **提炼** → 从记忆中提取有价值的学习内容
3. **推送** → 每天自动生成昨天学习卡片并发送给你
4. **复盘** → 每周自动生成上周学习周报和考题并发送给你

---

## 🎯 核心功能（5 个）

| 功能 | 说明 | 状态 |
|------|------|------|
| **1. 双渠道记忆读取** | 自动合并飞书 + WebUI 的每日记忆文件 | ✅ 已实现 |
| **2. 结构化数据提取** | 从记忆文件中提取主题/决策/踩坑/概念/金句 | ✅ 已实现 |
| **3. 学习卡片生成** | 生成 Markdown 格式的学习卡片（含进度条可视化） | ✅ 已实现 |
| **4. 自动定时推送** | 每天 10:00 自动生成并发送到飞书 | ✅ 已启用 |
| **5. 防重复机制** | 检查已发送标记，避免重复生成 | ✅ 已实现 |

## 🔧 提取的数据类型（5 类）

| 类型 | 识别规则 | 输出字段 |
|------|---------|---------|
| **📚 主题** | `## 新主题：{名称}` | 标题、来源渠道 |
| **📋 决策** | `| 选项 | 考虑因素 | 结果 |` 表格 | 选项、因素、结果 |
| **⚠️ 踩坑** | `- **问题：**` + `解决方案` + `教训` | 问题、方案、教训 |
| **🧠 概念** | `- **术语：**` + `定义` + `应用场景` | 术语、定义、场景 |
| **💬 金句** | `> "内容"` 引用格式 | 金句原文 |

## ⚙️ 配置项（用户可定制）

**配置文件：** `~/.daily-learning-cards/config.json`

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `extract.maxTopics` | 20 | 最大主题数 |
| `extract.maxPitfalls` | 6 | 最大踩坑数 |
| `extract.maxConcepts` | 6 | 最大概念数 |
| `extract.maxInsights` | 6 | 最大金句数 |
| `extract.minInsightLength` | 10 | 金句/洞见的最小字符长度 |
| `schedule.hour` | 10 | 定时小时 |
| `schedule.minute` | 30 | 定时分钟 |
| `language` | zh | 语言（zh/en/bilingual） |
